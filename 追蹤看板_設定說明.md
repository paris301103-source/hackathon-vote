# 北祥黑客松 2026 · 專案追蹤系統 — 設定說明（v2）

單檔 `追蹤看板.html`，沿用你現有的 Firebase 專案 `beixiang-hackathon`（資料存 `track`／`treports` 集合，與評審系統不衝突）。手機可用（RWD）。

## 一、Firestore 規則（加兩行）
到 Firebase → Firestore → **規則**，在 `match /databases/{database}/documents {` 裡加這兩行，按發布：

```
match /track/{id}     { allow read, write: if request.auth != null; }
match /treports/{id}  { allow read, write: if request.auth != null; }
```

（`control` 你原本已有，涵蓋權限與密碼設定，不用再加。）

## 二、四種角色（分層 + PUNK 分支）
登入頁用下拉選單選身分，再輸密碼。預設密碼：主管 `0000`、其他一律 `8888`（登入後可改）。

| 層級 | 誰 | 能做什麼 |
|---|---|---|
| 主管 / Leadership | 主辦 | 看全部、逐案檢視、本週會議、匯出、**設定每案權限** |
| 專案管理（負責人） | RAE·IAN·GIBBY／WIND／Allen／DIU／Julia／Bryan／Jaguar／Andy／Justin | 填自己負責的案、每週送出、**可叫停/暫停** |
| 合作者（隊長） | 各專案隊長（23 位） | 填自己專案的進度回報 |
| PUNK（資安/權限） | PUNK | 唯讀看全部，**每階段確認資安與權限**（打勾簽核） |

- 權限預設照分工帶（負責人＋隊長可輸入）；主管在「🔑 權限設定」頁可逐案調整「可輸入」名單。
- 要新增/調整帳號或分工，改 HTML 最上面的 `PROJECTS`（`pm`、`lead`）與 `MANAGERS`。

## 三、每個專案（一頁一個）
- **導入決策**：繼續導入／暫停觀察／叫停·不導入（任何階段都能改，附原因）。
- 目前進度（%＋狀態）、階段里程碑（7/8/9/10/Q4）。
- **PUNK 各階段資安/權限確認**（PUNK 打勾）。
- ✍️ 本週做了什麼、➡️ 接下來要做什麼、🚧 卡關、預期完成日。
- **📤 送出本週回報**：填完按送出才記錄成該週快照；主管即時看到。
- **📜 歷週紀錄**：回看過去每一週送出的內容。

## 四、主管檢視
- **總覽**：本週已送出 X/24、平均進度、進行中/卡關/完成/已叫停，可篩選的全案表格。
- **逐案檢視**：一頁一個，開會逐案走。
- **本週會議**：自動整理「還沒送出／卡關／暫停叫停／本週送出內容」。
- **權限設定**：逐案設定可輸入名單。
- **匯出 CSV**。

## 五、上線（Vercel）
建議把檔案改名 `index.html`（內容不用動），push 上 GitHub → Vercel 部署；或用 Netlify Drop 拖檔即得網址。手機開也 OK。

## 節奏
Top Priority：合作者/負責人**每週送出**、主管每週 leadership meeting 用「本週會議」頁檢視卡關、決策與產出。


## 六、權限與帳號管理（主管後台 · 「🔑 權限設定」頁）
- **調整每案角色**：可改每個專案的**負責人**、**可輸入（editors）**、**可檢視（viewers）**。
- **指定名單外的人**：按「＋ 指定他人」輸入名字即可；他用「**名字 ＋ 8888**」登入。
- **首次登入強制改密碼**：任何人第一次用預設密碼登入，都要先設定自己的新密碼才能進入。
- **忘記密碼 · 重設**：主管在同頁可把任一帳號密碼**重設回 8888**（對方下次登入再設新密碼）。綠點＝已自訂、灰點＝仍為預設。
